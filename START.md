# 项目介绍
创建一个一体化的前后端分离的通用后台管理后台框架项目
- 前端后端区分目录结构，前端项目在`/frontend`目录，后端项目在`/backend`目录。
    - 框架功能代码与用户额外业务代码使用不同的目录区分，即为用户提供单独的业务目录。方便用户更新框架。
- 对于后端，分为两个项目
    - 一个是用于实现maven包供多个用户进行复用，项目在`/backend/framework`目录，可发布到maven仓库或mvn install到本地仓库。
    - 另一个是用于实现具体的业务功能，通过maven引用framework到项目中，用于存储不同项目独特的业务逻辑，项目在`/backend/app`目录。
    - 框架功能全部写在framework项目中，用户业务逻辑写在app项目中。framework项目提供必要的自动配置，并提供给app项目配置文件，用户可以根据需要进行自定义配置。
- 项目的代码结构清晰，不过度封装，代码易读，有注释。
- 各功能模块独立，目录结构清晰，可插拔，也方便后期增加新的功能模块。可通过后台配置接口启用或禁用

# 功能列表
- 系统管理
    - 用户管理
    - 角色管理
        - 权限管理：为角色分配权限和操作
    - 部门管理
    - 菜单管理
    - 系统配置
        - 系统名称
        - 系统版本
        - 启用的模块
        - 系统默认首页
- 日志管理
    - 操作日志管理
    - 登录日志管理
- 字典管理
- 定时任务管理
- 图片管理

# 前端要求
- 基于Vue3，使用Composition API
- 使用TypeScript
- 使用element-plus
- 路由基于Vue Router实现
- 状态管理基于Pinia实现
- 与后端交互基于Axios实现
- 使用vite构建
- 使用VueUse，减少重复造轮子
- 项目既可以作为Web应用运行，也可以基于Electron打包，实现桌面端应用

# 后端要求
- 基于Spring Boot 3实现，版本3.5.11
- 数据库基于MySQL实现
- 数据访问基于MyBatis-Plus实现，使用mybatis-plus提供的方法，而不是mapper.xml或sql语句
- Mybatis-Plus的maven依赖名为mybatis-plus-spring-boot3-starter，版本3.5.15
- 项目基于Maven构建
- 集成Spring Security实现权限管理，支持菜单级权限、操作级权限、数据级权限
- 使用JWT实现无状态认证
- 提供RESTful API接口
- 良好的异常处理机制，返回统一的错误格式
- 支持分页查询
- 支持多数据源配置
- 保留数据库建表脚本，方便初始化数据库
- 集成Quartz实现定时任务管理，并可通过页面配置定时任务
